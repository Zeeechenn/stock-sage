# StockSage 项目工作约定

## 项目定位
StockSage 是 A 股个人辅助决策工具，目标是给建设性意见，不做自动化交易执行。

## 事实源优先级
每次做决策前优先读取：
1. `PROJECT.md` 项目进度
2. `PAPER_TRADING.md` 纸上交易状态
3. `stock-sage.db` 实际信号与长期标签
4. `backend/config.py` 默认参数
5. `ai_memory` 表中的关键记忆

## 行为约束
- 不预测价格，不鼓励强买。
- 单股仓位不超过配置上限，默认应向 15% 收敛。
- 单板块仓位不超过配置上限，默认应向 30% 收敛。
- 长期标签缺失时禁强买，最多给到买入。
- 测试 1/2 决策必须先声明规则版本。

## 记忆约定
应写入 `ai_memory`：
- 已买入股票、仓位、测试账户状态
- 风险预警和风控否决
- 测试规则切换
- 用户明确说“记住”的偏好或规则

不写入 `ai_memory`：
- 一次性查询
- 临时讨论
- 已经固定在 `backend/config.py` 的默认行为

## 启动检查清单
当用户说"项目s"或要求继续 StockSage 时：
1. 读 `PROJECT.md` 当前里程碑（M0–M6 体系，2026-05-15 起统一）。
2. 读取 active/global/test scope 的关键记忆。
3. 验证 `stock-sage.db` 可达。
4. 复述当前持仓和测试规则快照。

## 任务编号约定
所有新任务用 **M{X}.{Y}** 编号（里程碑 M0–M6 + 子任务）。历史"Phase / 阶段 / Tier / 执行计划"叫法在 `PROJECT.md` 顶部的术语映射表里有对应关系，新内容不再使用旧名。
