# StockSage 项目工作约定

## 项目定位
StockSage 是 A 股个人辅助决策工具，目标是给建设性意见，不做自动化交易执行。

## 事实源优先级
每次做决策前优先读取：
1. `PROJECT.md` 项目进度
2. `stock-sage.db` 实际信号与长期标签
3. `backend/config.py` 默认参数
4. `ai_memory` 表中的关键记忆

## 行为约束
- 不预测价格，不鼓励强买。
- 单股仓位不超过配置上限，默认应向 15% 收敛。
- 单板块仓位不超过配置上限，默认应向 30% 收敛。
- 长期标签缺失时禁强买，最多给到买入。
- 交易决策必须先声明规则版本。

## 记忆约定
应写入 `ai_memory`：
- 已买入股票、仓位、测试账户状态
- 风险预警和风控否决
- 测试规则切换
- 用户明确说“记住”的偏好或规则

不写入 `ai_memory`：
- 一次性查询
- 临时讨论
- 已经固定在 `backend/config.py` 的默认行为

## 时间约定
- 仅在**做交易决策、读取大盘行情、判断是否交易日**等需要精确时间的场景下运行 `date` 命令。
- 普通讨论、代码改动、文档更新等场景**不必**先读时间，避免不必要的命令。
- 时区说明：系统时区为 CST（China Standard Time, UTC+8），与新加坡 SGT（UTC+8）完全相同，时间数值无差异。
- 交易日判断以 A 股日历为准（周一至周五，非节假日）。

## 启动检查清单
当用户说"项目s"、"项目S"、"project s" 或要求继续 StockSage 时，固定指向 /path/to/stock-sage：
1. 读 `PROJECT.md` 当前里程碑（M0–M6 体系，2026-05-15 起统一）。
2. 读取 active/global/test scope 的关键记忆。
3. 验证 `stock-sage.db` 可达。
4. 复述当前持仓和规则快照。
5. 若涉及交易决策或大盘判断，再运行 `date`。

## 任务编号约定
所有新任务用 **M{X}.{Y}** 编号（里程碑 M0–M7 + 子任务，后续新方向顺延 M8+）。历史"Phase / 阶段 / Tier / 执行计划"叫法在 `PROJECT.md` 顶部的术语映射表里有对应关系，新内容不再使用旧名。若需要写项目规划，优先更新 `docs/ROADMAP.md` 的对应 M 编号段落；`PROJECT.md` 只做索引，`STATUS.md` 只放当前可操作快照，`CHANGELOG.md` 只记录已完成历史。不要为 StockSage 创建通用 `task_plan.md`、`progress.md`、`findings.md` 三件套。
