# Serenity 供应链瓶颈研究框架

> 方法，不是人格；研究，不是荐股。

这套框架蒸馏自公开的 Serenity 系列方法论（供应链瓶颈研究、证据分层、跨市场来源纪律、反确认偏误），**剥离了博主人格、收益神话和买卖话术**，改造成明仓 observe-only / source-gated / non-promoting 的研究模板。

**它和 [[a-teacher]] 的分工**：A老师框架回答"这个赛道方向对不对、现在能不能买"（海外领先指标 + 周期vs结构性 + 入场时机）；本框架回答"这条产业链里**哪一层是真瓶颈、证据够不够硬、什么能证伪它**"。两者互补，不重复。

**适用范围**：任何由需求冲击驱动的产业链主题（A 股为主，但来源纪律跨市场）。

**硬边界**（贯穿全程，不可违反）：
- 不输出买卖建议、目标价、仓位、预测涨跌幅、"强烈买入/确定上涨"等词。
- 不输出可被聚合成信号的分数；只输出 `research_priority_band` 档位。
- 社媒/KOL/传闻只能作为线索（lead），绝不作为唯一证据。
- 结论是"值不值得继续查 + 缺什么证据"，不是"该不该买"。

---

## 使用方式

输入一个主题/赛道/标的后，按以下六步完成，每步给出明确结论，不允许模糊带过。最终汇总成「输出模板」里的结构化字段。

---

## 第一步：瓶颈分层拆解（chain decomposition）

**核心问题：从下游需求一路拆到上游，哪一层在卡住系统产出？**

按这个链条逐层拆，每层问"这一层现在是不是瓶颈"：

```
下游需求 → 系统集成 → 芯片/器件 → 设备 → 材料 → 封测 → 基础设施
```

判定"真瓶颈层"的四个特征（越多越像真瓶颈）：

- **扩产慢**：建厂到量产 > 18 个月，单条线投资极高。
- **供应商少**：全球能做的就 2-3 家，集中度极高。
- **认证严**：客户认证周期 > 12 个月，切换成本高。
- **替代难**：材料/工艺/设备短期无国产或异路替代。

**结论格式**：给出 `chain_layers[]`（每层：层名 / 物理约束 / 已知供应商 / 证据等级 / 待答问题），并指出 `scarce_layer`（当前最稀缺的一层 + 证据）。

---

## 第二步：快速预筛（quick filter）

**在深入之前，先用四问筛掉"蹭概念"。四项全过才值得继续，否则降级。**

1. **forced demand（需求是否被硬约束推动）**：需求是由政策、架构迁移、客户 capex、不可逆工程约束推动，还是只有叙事？
2. **size mismatch（体量错配）**：下游需求规模 vs 候选公司当前收入/市值，是否有明显错位（错位才有弹性）？
3. **no substitute（无短期替代）**：是否存在短期难替代的材料/工艺/设备/认证/客户关系？
4. **outside voice（外部独立确认）**：近 90 天是否有 ≥3 个**独立**来源交叉确认（不是同一条新闻的转载）？

**结论格式**：`quick_filter_pass`（true/false）+ 未过项的原因。

> **分层记录（重要）**：当一个主题横跨多层、不同层答案不同时（例：设备层 no_substitute=✓、电解质层 no_substitute=✗），**必须按 chain_layer 分层记录 quick_filter，不要塌缩成主题级单一判定**。主题级 `quick_filter_pass` 取"最稀缺层"的判定；同时保留每层的 `forced_demand / size_mismatch / no_substitute / outside_voice` 明细。分层错位本身往往是最有价值的洞察（瓶颈到底卡在哪一层）。

---

## 第三步：证据分层（source tier）

**每条证据都要标来源等级。社媒只能是 lead。**

| 等级 | 含义 | 可用性 |
|---|---|---|
| `primary` | 一手：原始公告/招股书/问询函/电话会记录 | 最强 |
| `official` | 官方：交易所/监管/政府产业数据 | 强 |
| `filing` | 定期报告：年报/半年报/季报 | 强 |
| `ir` | 投资者关系：调研纪要、互动易回复（标注"公司回复≠审计事实"） | 中 |
| `industry` | 可信行业媒体/产业数据库/海外龙头披露 | 中 |
| `social_lead` | 社媒/KOL/传闻 | **仅 lead，不能作唯一证据** |

**结论格式**：`evidence_tier`（本论题最强证据落在哪一级）+ `source_refs[]`（带 locator：公告标题/日期/章节/页码/表格位置）。

---

## 第四步：A 股 source playbook（导入前的来源优先级）

研究 A 股标的时，按以下优先级找直接来源，从上往下逐级降级可信度：

1. CNINFO / 巨潮资讯公告
2. 上交所 / 深交所 / 北交所公告
3. 年报、半年报、季报
4. 投资者关系活动记录表
5. 互动易 / e 互动（**必须标注为"公司回复"，不等同审计事实**）
6. 再融资、重大合同、关联交易、股权激励、减持、限售解禁、质押公告
7. 政策文件、产业链官方数据
8. 可信行业媒体 / 产业数据库 / 海外龙头披露（领先指标）
9. 社媒、KOL、传闻 —— **仅作 lead**

**降级触发器**：
- 只有社媒/媒体叙事、无任何公告/财报/订单 → 证据等级直接 `social_lead`，论题降级。
- 关键证据日期晚于 `as_of` → 该证据作废（防 lookahead）。
- 近 90 天无新鲜直接来源 → 标 `needs_check`。

---

## 第五步：贝叶斯论题追踪（Bayesian update）

**从"发现一个好故事"转向"持续追踪一个可证伪假设"。** 每条论题记录：

- `prior_view`：初始判断，必须写成**可证伪**的假设。
- `new_evidence`：本轮新增证据摘要 + 来源等级。
- `posterior_direction`：`up` / `down` / `unchanged` / `unclear`。
- `confidence_delta`：`small` / `medium` / `large`（**不用收益数值**）。
- `what_would_change_view`：哪些后续证据会改变判断。
- `stale_after`：该证据/论题的过期时间。

---

## 第六步：反方先行 QA（bear-before-bull）

**反确认偏误：先写为什么这条论题是错的，再写为什么对。**

- 先写 `bear_case`：这条瓶颈论题最可能错在哪（替代技术、产能其实够、需求证伪、估值泡沫、治理风险）。
- 列出 **≥3 条**能推翻 thesis 的具体事实（`falsification_questions[]`）。
- 区分清楚：公司事实 / 行业判断 / 社媒线索，三者不混。
- 如果没有做独立复核，显式写明"未做独立复核"。

---

## research_priority_band（研究优先级档位，非分数）

综合六步，给出**档位**，只用于排序人工复核队列，**绝不是买卖信号、不是分数**：

| 档位 | 含义 |
|---|---|
| `够查` | 真瓶颈特征明显 + 证据等级 ≥ filing/official + quick filter 全过 + 证伪条件清晰 → 值得继续深查 |
| `暂缓` | 方向有意思但证据等级不足（停在 ir/industry）或 outside voice < 3 → 先补源再说 |
| `证据不足` | 只有 social_lead / 媒体叙事，或 quick filter 未过 → 不投入深查，仅记录线索 |

## 输出模板（对应 `SerenityChokepointReport` 字段）

```
## [主题/赛道/标的] — Serenity 瓶颈研究

### chokepoint_layer（一句话：瓶颈卡在哪一层）
[...]

### chain_layers（逐层表）
| 层 | 物理约束 | 已知供应商 | 证据等级 | 待答问题 |
|---|---|---|---|---|
[...]

### scarce_layer（最稀缺层 + 证据）
[...]

### quick_filter_pass（按 chain_layer 分层；主题级取最稀缺层）
| 层 | forced_demand | size_mismatch | no_substitute | outside_voice(90d≥3) |
|---|---|---|---|---|
| [层名] | [✓/✗ + 理由] | [✓/✗] | [✓/✗] | [✓/✗，列来源] |
主题级 quick_filter_pass: [取 scarce_layer 的判定 + 一句话] | 分层错位（如有）: [...]

### evidence_tier + source_refs
最强证据等级：[primary/official/filing/ir/industry/social_lead]
来源（带 locator）：[...]

### substitute_risk
[替代路径/绕路可能性，必填，防单边叙事]

### bayesian
prior → new_evidence → posterior_direction → confidence_delta → what_would_change_view → stale_after

### bear_case + falsification_questions（≥3）
反方：[...]
能推翻的事实：1. ... 2. ... 3. ...

### research_priority_band
[够查 / 暂缓 / 证据不足] + 一句话理由 + 还缺什么证据
```

---

## 重要约束（再次强调）

- **不接入长期标签聚合**：本框架的输出是独立研究报告，不走明仓 `LongTermTeam` 的 track/quality/boom/flow 加权，不影响官方信号、仓位、止盈止损。
- **不出买卖语言**：禁用"买入/卖出/加仓/减仓/目标价/仓位/火速上车/确定上涨/强烈推荐"等。
- **社媒只能 lead**：任何结论不能只靠社媒/传闻支撑。
- **证据晚于 as_of 一律作废**：防止用未来信息。
- **有反例必须说**：找到任何与论题矛盾的证据，必须呈现，不允许只选支持性证据。
- **结论永远是"值不值得继续查 + 缺什么"**，不是"该不该买"。
